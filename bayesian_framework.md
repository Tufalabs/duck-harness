- **Perception → abstraction, not raw grids.** Don't feed 64×64 integer arrays to the model. The graph-exploration work used image segmentation, status-bar detection and masking, and state hashing — segment each frame into objects (connected components, color, position), compute the *diff* between frame *t* and *t+1* after an action, and hash states. The diff is your core learning signal: it tells you what each action does. This is where A3B's vision encoder can pull its weight. arxiv
- **World model = explicit and executable.** The most promising recent direction is verifier-driven executable world models: the agent learns from limited interaction, compresses observations into a model, and uses that model to decide which real actions are worth spending — because direct experimentation is costly enough that blind trial-and-error is inadequate. In practice: have the LLM write a small Python model of the dynamics ("action 2 moves the blue object right unless a wall blocks it"), run it as a verifier against logged transitions, and have the LLM repair it when a prediction is falsified. The paper's honest caveat is that executable world models alone aren't sufficient — pair them with search. arXiv
- **State graph + informed exploration.** Build the directed (state, action)→state graph the runners-up used; track cycle detection, reachability, max depth, and whether the reachable graph is fully explored, and record invalid actions as no-change transitions. Prioritize untried/novel actions early (the "new-action prioritization" step gave a measurable lift), then plan a short path once the model is good enough. arXiv
- **Goal inference.** You're never told the objective; the agent must identify desirable future states from environmental cues and score/level-completion signals. Maintain goal hypotheses and test them. Level-complete and terminal states are explicitly marked, so those transitions are your supervision. arxiv
- **Cross-level memory.** Levels add mechanics. Carry the learned world model forward and frame each new level as "known model + detect the delta," not a cold start. ARC Prize frames the three target capabilities as modeling, goal-setting, and planning/execution with course-correction.
